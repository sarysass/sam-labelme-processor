# 基于 SAM 的线虫 Mask 后处理研究进展报告

## 1. 报告目的

本文总结本项目在“显微图像线虫 mask 后处理”方向上的阶段性研究进展，重点说明：

- 当前实际遇到的问题是什么
- 哪些方案被验证有效
- 哪些方案已经被尝试但应明确放弃
- 为什么当前问题不能只靠几何规则解决
- 下一阶段应如何设计

本文面向后续参与该项目的实习生与工程师，目标是让后来者快速理解：当前代码为什么这样写、哪些方向已经踩过坑、后续应该从哪里继续。

## 2. 项目背景

本项目的基本流程是：

1. 读取图像和边界框
2. 使用 SAM 生成目标 mask
3. 将结果保存为 Labelme 可编辑格式
4. 对生成的 mask 做后处理，降低人工精修成本

项目核心文件如下：

- 后处理核心：`src/core/mask_edge_optimizer.py`
- CLI：`optimize_mask_edges.py`
- 回归测试：`tests/test_mask_edge_optimizer.py`

研究的目标不是重新训练 SAM，而是在现有 SAM 输出基础上，尽可能通过后处理把 mask 修到“人工少量修正即可使用”的程度。

## 3. 当前数据上的核心问题

这一批显微图像上的典型问题不是“目标完全分不出来”，而是“拓扑结构错误”。

更具体地说，常见现象是：

- 线虫主体大致被分出来了
- 卷曲结构内部本应是背景的区域，被 SAM 合并成了前景
- 即使把内部空腔重新挖出来，空腔外侧仍然可能残留一层多余的 mask
- 这层残余区域通常较薄，但可能较长，会沿卷曲结构延伸

因此，问题不能简单理解成“边缘贴合不够”，而应理解成两类问题叠加：

- 边界级问题：边缘不够贴合、局部锯齿、轮廓偏移
- 拓扑级问题：卷曲内腔被填死、空腔外侧残余背景壳层未被去掉

## 4. 数据特点与算法难点

这批图像具有几个显著特征：

- 虫体与背景灰度差距不大
- 图像边界弱，梯度不稳定
- 背景中存在阴影、晕圈和纹理
- 线虫经常卷曲，自身相邻部分距离非常近
- 目标局部会形成封闭或半封闭结构

这意味着：

- 纯边缘检测容易被弱边界和背景纹理干扰
- 纯形态学规则容易把细虫体误判成错误连接
- 纯几何外推容易“继续扩大空腔”，而不是真正删除外侧残余背景

## 5. 当前稳定有效的方案

### 5.1 基础边缘优化

项目中当前稳定保留的第一层后处理，是局部 edge-guided refinement。

核心思路：

1. 只在当前 mask 附近取局部 patch
2. 对 patch 灰度化并做轻量平滑
3. 使用 Canny 获取候选边缘
4. 结合当前 mask 做窄带 watershed 微调
5. 最后做轻量平滑，减少锯齿

它的优点是：

- 对普通轮廓偏移有效
- 对一般边界贴合问题有效
- 对小目标边缘的局部修正有效

它的局限是：

- 它只能修边界
- 不能主动修复“内部背景被并成前景”的拓扑错误

结论：

- 这一步是有价值的
- 但它不是当前研究问题的核心解法

### 5.2 内部空腔恢复（cavity recovery）

这是当前研究中最重要、也是已经被真实样本验证有效的一步。

核心思路：

1. 在当前 mask 内部寻找候选背景区域
2. 候选区域必须离 mask 外边界有一定距离
3. 候选区域的像素分布要更像外部背景，而不像虫体主体
4. 删除候选区域后，主体连通性不能被破坏

这个设计故意非常保守，它只尝试恢复“封闭在 mask 内部的背景块”。

优点：

- 能有效恢复卷曲线虫中心被错误填满的空腔
- 对真实问题有明确改善
- 风险相对可控

局限：

- 它只修复“内部封闭背景”
- 一旦某块背景与外部可连通，它就不会处理
- 因此会留下空腔外侧的一圈残余 mask

结论：

- cavity recovery 是当前必须保留的核心成果
- 它解决了问题的一半，但不是完整解法

## 6. 为什么当前 cavity recovery 只能挖内部空腔

这是整个研究过程中一个非常关键的认识点。

当前 cavity recovery 之所以只能挖“内部空腔”，不是因为实现不够强，而是因为它的目标就是“保守恢复封闭内腔”。

当前逻辑有两条核心约束：

1. 候选点必须位于 mask 内部，且离 mask 外边界有一定距离
2. 候选块不能与 mask 外部直接接触

这两条约束带来的直接后果是：

- 内部洞可以被恢复
- 和外部可连通的背景区域一律不会被删

这样做的好处是：

- 不容易沿着弱边界把虫体主体误删掉
- 在灰度接近、边界模糊的数据上更安全

坏处是：

- 不能处理空腔外侧残余壳层
- 会留下一圈多余的背景区域

因此，第一阶段的保守性是有意义的，但也决定了它无法覆盖第二类问题。

## 7. 已尝试但应明确放弃的方案

在本轮研究中，我们尝试过多种针对“卷曲内腔与残余连接”的修正方案。结果表明，下面这些方向不应继续作为主路径维护。

### 7.1 窄桥去除（bridge cut）

思路：

- 找出 mask 中局部很窄的连接
- 将其视作“假桥”并切断

问题：

- 对非常窄的错误连接有时有效
- 对大面积被糊住的内腔无效
- 容易误伤本来就细的真实虫体部分

结论：

- 不适合这类数据
- 已从主代码路径中移除

### 7.2 path cut / cavity path cut

思路：

- 从已恢复的空腔出发
- 在当前前景中寻找一条“更像背景”的路径
- 沿这条路径切开连接

问题：

- 它更像是在“切一刀”
- 适合单条错误连接
- 不适合“空腔外侧残余壳层”这种整片区域性问题

结论：

- 对当前主问题打击面不对
- 已移除

### 7.3 multi-cut path cut

思路：

- 在 path cut 上扩展到多个切口

问题：

- 理论上更强
- 实际在关键真实样本上改善有限
- 仍然没有抓住“壳层残余”的本质

结论：

- 不继续保留

### 7.4 normal-guided cavity expansion

思路：

- 以已恢复出的空腔边界为起点
- 沿法线方向向外推进
- 试图把空腔边界“推到”真实边界

问题：

- 在 synthetic case 上可以构造成功样例
- 但对真实样本 `0988-5` 的关键实例 `group_id 7/8` 没有稳定收益
- 真实图像中的弱边界、低对比度和复杂卷曲结构使这种局部法线外推不稳

结论：

- 合成数据通过不代表真实有效
- 不应继续沿这个方向投入

## 8. 为什么纯几何方法不够

在这轮研究中，一个非常明确的结论是：

**纯几何规则只能做候选筛选，不能做最终判定。**

所谓纯几何规则，包括：

- 细不细
- 薄不薄
- 靠不靠洞
- 靠不靠边界
- 删除后是否破坏连通性

这些条件都很重要，但它们只能回答：

- “哪里值得看”

它们不能稳定回答：

- “这块到底是背景还是虫体”

原因很简单：

- 很薄的区域，可能是真虫体
- 靠近空腔的区域，可能仍属于前景主体
- 不看图像内容，只看 mask 形状，容易把细虫体误删掉

因此，第二阶段必须引入真实的前景/背景判别，而不是继续堆几何规则。

## 9. 当前最新原型：cavity growth

在当前工作区里，我们已经写了一个新的实验原型，暂时称为 `cavity growth`。

它的目标是：

- 以已恢复出的空腔为种子
- 向外逐层生长
- 试图吃掉紧贴空腔的薄残余前景

它在 synthetic case 上是成立的，说明“第二阶段需要存在”这件事没有问题。

但在真实样本 `0988-5` 上，这一版仍然没有达到预期。

原因经过验证已经很明确：

- 它去掉的主要仍然是“靠近已挖空腔的面积”
- 而不是“空腔外侧那一层真正多余的背景壳层”

也就是说，它目前本质上更像：

- “继续扩大空腔”

而不是：

- “剥离空腔外侧残余背景壳层”

这说明：

- 第二阶段这个位置是需要的
- 但当前这版 `cavity growth` 不是正确实现

## 10. 下一阶段的正确设计方向

下一阶段应该设计成：

### 第一阶段：cavity recovery

继续保留当前已经稳定的“封闭内腔恢复”。

### 第二阶段：background-like shell removal

第二阶段不再是：

- 再扩一次洞
- 再切一刀
- 再做法线推进

而应该是：

1. 先提取“候选壳层”
2. 再做真实的前景/背景判别
3. 最后只删除高置信的背景壳层组件

### 第二阶段候选壳层的几何定义

候选壳层应满足：

- 位于当前 mask 内
- 紧贴已恢复空腔
- 局部较薄
- 可以很长，不要求必须短
- 删除后主体仍保持连通

这里特别强调一点：

- 候选区域可以很长
- 但一定要薄

这和前面被否掉的“桥”“路径”“短切口”抽象不同。

当前问题更像：

- 一层沿卷曲结构延伸的薄背景壳层

而不是：

- 一条短桥或一条短连接

### 第二阶段的图像判别

对候选壳层连通块，不应该只看几何，而应计算一个背景分数。

建议第一版分数综合三类特征：

- 与外部背景的相似度
- 与虫体主体的差异度
- 局部梯度/边界是否支持这里是一块背景带，而不是实心虫体

也就是说：

- 几何条件决定“哪里可以看”
- 图像判别决定“那里该不该删”

## 11. 当前代码状态说明

截至当前，代码状态应理解为两部分：

### 已稳定、可保留

- 基础边缘优化
- cavity recovery
- 通用辅助函数
- 已验证测试框架

### 尚属研究原型，不应视为定型方案

- cavity growth

换句话说：

- 当前工作区的第二阶段代码不能直接视作最终方案

## 12. 工程方法论总结

这轮研究中，有几条经验对实习生尤其重要。

### 12.1 方向正确，不等于实现已经正确

例如：

- 第二阶段必须存在，这个判断是对的
- 但当前 `cavity growth` 的实现方式仍然是错的

不要把“模块位置对”误认为“算法已经对”。

### 12.2 synthetic 通过，不等于真实数据有效

很多实验方向在构造样例上都可以成立，但真实显微图像的分布更复杂。

因此：

- 合成测试用于验证机制是否存在
- 真实样本验证用于判断机制是否有工程价值

后者优先级更高。

### 12.3 应先把问题分类，再做算法

当前至少已经识别出三类不同问题：

- 边界贴合不够
- 内部空腔被填死
- 空腔外侧薄壳残余未被去掉

这三类问题不能用同一种规则硬解。

### 12.4 失败实验必须及时清理

我们已经将一批无效实验分支从主代码路径中清理出去。这一步非常重要。

原因是：

- 如果失败方向仍然留在主路径中
- 后来的维护者会误以为它们是“可选有效工具”
- 这会制造大量理解成本和错误路径依赖

## 13. 当前阶段性结论

截至目前，可以形成如下阶段性结论：

1. SAM 输出可以作为高价值起点，但不能直接作为最终标注
2. 基础边缘优化对普通轮廓修正有效，但无法解决拓扑错误
3. cavity recovery 是当前最有效、最稳定的核心成果
4. bridge cut、path cut、multi-cut、normal-guided 等实验方向不适合继续保留
5. 纯几何方法不能解决“空腔外侧残余背景壳层”问题
6. 第二阶段必须引入真实前景/背景判别
7. 当前 `cavity growth` 只是第二阶段的原型，不是最终解法

## 14. 推荐后续研发顺序

建议后续严格按以下顺序推进：

1. 保持当前稳定基线不动
2. 为“空腔外侧残余壳层”设计新的 synthetic 测试和真实样本回归
3. 单独实现 `background-like shell removal`
4. 在真实关键样本上验证是否确实删除的是“外侧残余壳层”而不是“已挖空腔面积”
5. 验证通过后，再考虑小批量推广
